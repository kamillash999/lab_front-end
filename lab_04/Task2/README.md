# Лабораторная работа 4.2: Рефакторинг менеджера статей

## Описание рефакторинга
Целью данной работы был рефакторинг монолитного компонента `ArticleManager` в систему мелких, многоразовых компонентов для улучшения читаемости и производительности кода

## Распределение ответственности между компонентами:
ArticleManager (Контейнер): Хранит основной массив статей и бизнес-логику добавления/удаления элементов
AddArticle: Изолированный компонент формы, который получает данные и обработчики событий через `props`
ArticleList: Отвечает исключительно за итерацию по массиву данных и рендеринг списка дочерних компонентов
ArticleItem: Управляет собственным состоянием раскрытия содержания (`isOpened`) с помощью локального `useState. то позволяет избежать лишних рендров всего списка при взаимодействии с одно статье.
