# Лабораторная работа №7: Оптимизация и Code Splitting

## Описание работы
В этой лабораторной я научилась делать так, чтобы приложение загружалось быстрее за счет разделения кода. Теперь тяжелые компоненты подгружаются только тогда, когда они действительно нужны пользователю.

## Что я сделала:

### 1. Ленивая загрузка (Task 1)
Я реализовала динамический импорт компонентов.
* **Как это работает:** Использовала `React.lazy` для "тяжелых" блоков. 
* **Suspense:** Добавила компонент `Suspense`, который показывает красивый индикатор загрузки (spinner), пока браузер скачивает нужную часть кода.
* **Результат:** Основной размер приложения уменьшился, так как код подгружается по частям.

### 2. Предохранители — Error Boundaries (Task 2)
Я создала специальный компонент, который ловит ошибки в коде.
* **Функционал:** Если в каком-то блоке произойдет сбой, всё остальное приложение продолжит работать, а пользователь увидит вежливое сообщение об ошибке вместо "белого экрана".
* **Особенность:** Разобралась, как работают методы жизненного цикла `getDerivedStateFromError` и `componentDidCatch`.

## Чему я научилась:
- Использовать `Code Splitting` для ускорения первой загрузки сайта.
- Настраивать плавные переходы между состояниями загрузки с помощью `Suspense`.
- Изолировать ошибки в компонентах, чтобы приложение было стабильным.
